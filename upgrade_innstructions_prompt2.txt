Additional requirement: Harden AI response handling and validation.

Background:
Previously the application failed when `aiResponseFarmer` was not returned as a plain string. In some cases the AI returned an object, array, or malformed structure which caused dashboard rendering and API processing errors.

Implement robust validation for all incoming case payloads.

Requirements:

1. Validate request payload before saving.

Required fields:

* caseId
* phoneNo
* imageUrl
* aiResponseFarmer
* aiResponseDashboard
* status

2. Enforce data types:

```ts
caseId: string
phoneNo: string
imageUrl: string
aiResponseFarmer: string
status: string
```

3. Handle aiResponseDashboard safely:

* If already a string, store as-is.
* If object, serialize using JSON.stringify().
* If null/undefined, store empty string.

4. Strict validation:

Reject payload if:

```ts
typeof aiResponseFarmer !== "string"
```

Return HTTP 400 with clear error message.

5. Add runtime validation helper:

```ts
validateCasePayload(payload)
```

6. Prevent React rendering crashes:

* Never render raw objects.
* Ensure dashboard pages gracefully handle malformed or missing fields.
* Add fallback values when fields are missing.

7. Add defensive API logging:

* Log invalid payloads.
* Log validation failures.
* Log database insertion failures.

8. Add unit tests covering:

* Valid payload
* Missing fields
* aiResponseFarmer as object
* aiResponseFarmer as array
* aiResponseDashboard as object
* aiResponseDashboard as string

Goal:
The application should never crash due to malformed AI responses. Invalid payloads should be rejected with proper validation errors before reaching the database or UI.
