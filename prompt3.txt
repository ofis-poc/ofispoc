# Prompt

You are continuing the OFIS AgriConnect project.

Do NOT redesign the existing UI.

Do NOT change the existing Survey Builder architecture.

Work only on the next module after Survey Publishing.

## Objective

Implement a Survey Assignment module.

When an admin clicks **Publish Survey**, they should be able to select farmers from the existing database and assign the survey to them.

After assignment, the system should trigger an n8n webhook which will handle the WhatsApp survey flow.

## Create New Table

Create a PostgreSQL table named

survey_assignments

Schema:

```sql
CREATE TABLE "ofis-farmer".survey_assignments (

    id SERIAL PRIMARY KEY,

    survey_id VARCHAR(50) NOT NULL,

    phone_no VARCHAR(30) NOT NULL,

    assigned_at TIMESTAMP DEFAULT NOW(),

    status VARCHAR(30) DEFAULT 'PENDING'
);
```

Status values:

* PENDING
* IN_PROGRESS
* COMPLETED
* FAILED

Do NOT create dynamic tables.

Every survey assignment must be stored in this single table.

## Trigger n8n

After successful inserts

POST

to

POST /webhook/start-survey

Payload:

```json
{
  "surveyId": "<survey_id>",
  "assignmentTable": "survey_assignments",
  "responseTable": "survey_responses"
}
```

Do NOT wait for n8n completion.

Fire-and-forget request.

If webhook fails

show toast

"Survey published but webhook trigger failed."
