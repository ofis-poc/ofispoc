We need to migrate our Next.js dashboard application from local `db.json` storage to PostgreSQL while preserving all existing functionality and UI behavior.

Current state:

* Next.js application deployed on Vercel.
* Cases are currently stored and read from `db.json`.
* APIs:

  * GET `/api/cases`
  * GET `/api/cases/[id]`
  * POST `/api/cases`
  * POST `/api/cases/[id]/resolve`
  * GET `/api/stats`
* Dashboard contains:

  * Case Queue page
  * Case Details page
  * Expert Review workflow
  * Status tracking (HIGH_CONFIDENCE, EXPERT_REVIEW_REQUIRED, RESOLVED)

Goal:
Replace all `db.json` reads/writes with PostgreSQL while keeping API contracts unchanged so the frontend requires minimal or no modifications.

Requirements:

1. Create PostgreSQL schema:

```sql
CREATE TABLE cases (
    id SERIAL PRIMARY KEY,
    case_id VARCHAR(50) UNIQUE NOT NULL,
    phone_no VARCHAR(30),
    image_url TEXT,
    ai_response_farmer TEXT,
    ai_response_dashboard TEXT,
    status VARCHAR(50),
    expert_diagnosis TEXT,
    expert_recommendation TEXT,
    created_at TIMESTAMP,
    resolved_at TIMESTAMP
);
```

2. Use modern PostgreSQL integration suitable for Vercel.

   * Prefer Neon PostgreSQL.
   * Use environment variable:
     DATABASE_URL

3. Create a reusable database module:

   * Connection pooling
   * Query helper functions
   * Proper error handling

4. Refactor APIs:

   * Replace file operations on db.json with SQL queries.
   * Preserve existing request/response formats.
   * Keep all current endpoints functional.

5. Implement:

   * Create Case
   * Get All Cases
   * Get Case By ID
   * Resolve Case
   * Dashboard Statistics

6. Add migration script:

   * Read existing db.json
   * Insert all existing records into PostgreSQL
   * Avoid duplicates based on case_id

7. Add TypeScript types/interfaces for:

   * Case
   * CreateCaseRequest
   * ResolveCaseRequest

8. Update project structure cleanly:

   * lib/db.ts
   * lib/cases.ts
   * migrations/
   * types/

9. Keep backward compatibility:

   * Existing frontend pages should continue working without UI changes.

10. Generate:

* SQL schema
* Database helper code
* Updated API route implementations
* Migration script
* Environment variable examples
* Step-by-step deployment instructions for Vercel + Neon.

Please analyze the current project structure and implement the complete migration end-to-end with production-quality code.
